# План: Переход к восстановлению вектора поляризации P

## Физическая постановка

### Текущий подход
Нейросеть предсказывает напрямую:
- ρ (плотность заряда) — выход 5
- j (плотность тока) — выходы 6-8

### Новый подход
Нейросеть предсказывает вектор поляризации **P = (Px, Py, Pz)**:
- **ρ = -div P = -(∂Px/∂x + ∂Py/∂y + ∂Pz/∂z)**
- **j = ∂P/∂t = (∂Px/∂t, ∂Py/∂t, ∂Pz/∂t)**

### Уравнения Максвелла в терминах P
Волновые уравнения остаются теми же, меняются только источники:

1. **□φ = -4πρ/ε = 4π·div P / ε**
2. **□A_i = -4πμ·j_i/c = -4πμ/c · ∂P_i/∂t**
3. **∇·A + (εμ/c)·∂φ/∂t = 0** (калибровка Лоренца — без изменений)

#### 1.1. Структура `VariableSet`

- **Добавить производные P**: `DP_dt`, `DP_dx`, `DP_dy`, `DP_dz` (по аналогии с A)

В конструкторе `VariableSet`:
- Заменить `@variables ρ(..), jx(..), jy(..), jz(..)` на `@variables Px(..), Py(..), Pz(..)`

#### 1.2. Функция `create_derivative_equations`

#### 1.3. Функция `create_pde_system`
**Ключевые изменения в уравнениях:**

Уравнение для φ:
```julia

# Стало: ~ 4 * pi * (Dx(Px) + Dy(Py) + Dz(Pz)) / constants.ε
```
где Dx(Px) = Differential(x)(variables.P[1](coords...)) и т.д.

Уравнения для A_i:
```julia
# Было: ~ -constants.μ * 4 * pi / constants.c * variables.j[i](coords...)
# Стало: ~ -constants.μ * 4 * pi / constants.c * variables.DP_dt[i](coords...)
```

Список переменных `allvars`:
- **Без производных**: [φ, Ax, Ay, Az, Px, Py, Pz] → **7 переменных**
- **С производными**: 7 + 4 (φ) + 12 (A) + 12 (P) → **35 переменных**

#### 1.4. Функция `create_boundary_conditions`
**Удалить** граничные условия для ρ и j (их нет и сейчас — только φ и A).
**Добавить** опциональные граничные условия для P (если нужно, чтобы P = 0 на границе).

#### 2.3. Функция `create_constrained_neural_network`
**Текущее поведение:** Умножает ρ (индекс 5) на head_indicator.

**Новое поведение:** Умножает **все компоненты P** (индексы 5-7) на head_indicator.

Это гарантирует P = 0 вне эллипсоида → div P = 0 → ρ = 0 вне головы.

```julia

rho_constrained = rho .* indicator

# Стало:
P = output[5:7]
P_constrained = P .* indicator  # broadcast на все 3 компоненты

output_constrained = vcat(
    output[1:4],
    P_constrained,
    output[8:end]  # если есть производные
)
```

### 3. `src/neural_pde_solver/Optimization.jl`

#### 3.1. `compute_field_energy_loss`
**Текущая формула:** E_field = ∫(ρ·φ + A·j) dV

**Новая формула** (через P):
```
E_field = ∫((-div P)·φ + A·∂P/∂t) dV
```

Реализация:
- ρ вычисляется как -(∂Px/∂x + ∂Py/∂y + ∂Pz/∂z) через конечные разности
- j = ∂P/∂t берётся из сети (DP_dt) или через конечные разности по времени

**Альтернатива** (интегрирование по частям):
```
E_field = ∫(P·∇φ + A·∂P/∂t) dV  (без поверхностных членов, т.к. P=0 на границе)
```
Это может быть численно стабильнее, т.к. не требует вычисления div P.

#### 3.2. `compute_tv_regularization`
**Текущее поведение:** TV применяется к ρ (индекс 5).

**Новое поведение:** TV применяется к **div P** (т.е. к ρ = -div P).

Реализация:
1. Вычислить ρ = -(DPxDx + DPyDy + DPzDz) через конечные разности
2. Применить TV к ρ как раньше

**Альтернатива:** TV на компоненты P напрямую (менее физично, но проще).

#### 3.3. `data_loss` и `derivative_loss`
**Без изменений** — они работают только с φ, который не меняется.

#### 3.4. `create_additional_loss`
- Обновить вызов `compute_field_energy_loss` (новая сигнатура)
- Обновить вызов `compute_tv_regularization` (новая логика)

## Риски и замечания

1. **Численная стабильность**: Вычисление div P через конечные разности может быть менее стабильным, чем прямое предсказание ρ. Рекомендуется использовать предсказанные производные (use_derivatives=true).

2. **Размерность выхода**: 35 переменных вместо 24 — увеличит время обучения и потребление памяти.

3. **TV регуляризация**: TV на div P требует двойного дифференцирования (P → div P → grad(div P)), что может быть нестабильно. Рассмотреть TV на самих компонентах P.

4. **Совместимость**: Старые .jld2 файлы с результатами будут несовместимы с новой структурой.
