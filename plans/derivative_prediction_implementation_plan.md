# План реализации предсказания производных для улучшения сходимости

## Обзор проблемы

Текущая реализация использует автоматическое дифференцирование второго порядка для расчета оператора Даламбера, что приводит к высокой чувствительности к шуму и медленной сходимости. Предлагаемый подход - предсказывать напрямую производные первого порядка через нейросеть, что уменьшит шум и улучшит сходимость.

## Ключевые изменения в архитектуре

### Текущая архитектура (8 выходов):
```
[φ, Ax, Ay, Az, ρ, jx, jy, jz]
```

### Новая архитектура (24 выхода):
```
[φ, Ax, Ay, Az, ρ, jx, jy, jz,
 DφDt, DφDx, DφDy, DφDz,
 DAxDt, DAxDx, DAxDy, DAxDz,
 DAyDt, DAyDx, DAyDy, DAyDz,
 DAzDt, DAzDx, DAzDy, DAzDz]
```

## Подробное описание изменений

### 1. PDEDefinitions.jl

#### 1.1. VariableSet
- Добавить поля для производных φ (DφDt, DφDx, DφDy, DφDz)
- Добавить поля для производных Ax, Ay, Az (каждая компонента имеет 4 производных)
- Обновить конструктор для создания всех переменных при инициализации

#### 1.2. create_variables()
- Изменить функцию для создания 24 переменных вместо 8
- Добавить объявление @variables для всех производных

#### 1.3. dalembert_operator()
- Заменить расчет ∂²φ/∂t² через autodiff на ∂(DφDt)/∂t
- Заменить расчет ∂²φ/∂x² на ∂(DφDx)/∂x и так далее для y, z
- Оставить для PML член γ*∂φ/∂t расчет через autodiff на φ
- Новая формула оператора:
  ```julia
  Δ = Differential(x)(DφDx) + Differential(y)(DφDy) + Differential(z)(DφDz)
  d2F_dt2 = Differential(t)(DφDt)
  pml_term = γ * Differential(t)(φ)  # γ*∂φ/∂t через autodiff
  ```

#### 1.4. create_pde_system()
- Добавить все 24 переменные в PDESystem
- Добавить связывающие уравнения для производных:
  ```julia
  Dt(φ) ~ DφDt
  Dx(φ) ~ DφDx
  Dy(φ) ~ DφDy
  Dz(φ) ~ DφDz
  ```
- Аналогично для Ax, Ay, Az
